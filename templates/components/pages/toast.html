Levels

Each button pushes a toast directly via window.apexToast(). Toasts auto-dismiss after 5 seconds.

With action

Add a button that links somewhere — useful for "Undo" or "View" follow-ups.

Persistent

Persistent toasts don't auto-dismiss — user must click the close button. Use sparingly, only for things that require attention.

Server-side usage

Push from any view via the messages bridge — toast renders on the next request.

from apps.core.messages import toast, LEVEL_SUCCESS

def my_view(request):
    ...
    toast(request, LEVEL_SUCCESS, "Saved.",
          action={"label": "View", "url": obj.get_absolute_url()})
    return redirect("…")

# Or use plain Django messages — they light up too:
from django.contrib import messages
messages.success(request, "Saved.")

Client-side usage

// Available globally on any page where the toast container is rendered.
apexToast({
  level: 'success',          // success | info | warning | error
  body: 'Saved.',
  action: { label: 'View', url: '/foo/1/' },  // optional
  persistent: false,         // optional, default false
});

Accessibility

  • • Container has aria-live="polite" + aria-atomic="false".
  • • Each toast has role="status".
  • • Dismiss button has aria-label="Dismiss".
  • • Auto-dismiss extends to 8 seconds when prefers-reduced-motion is set.
