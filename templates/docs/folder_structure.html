Inside an app

Each apps/<name>/ follows the same shape — predictable to navigate after seeing one or two:

{% code lang="text" %}apps/customers/ ├── __init__.py ├── apps.py # AppConfig ├── models.py # ORM models ├── views.py # Class-based views (ListView, DetailView, etc.) ├── forms.py # ModelForm subclasses with our widget overrides ├── urls.py # app_name + urlpatterns ├── admin.py # Django admin registrations (optional) ├── signals.py # Signal handlers (optional — see activity/) ├── migrations/ │ └── 0001_initial.py ├── templates/customers/ │ ├── customer_list.html │ ├── customer_detail.html │ └── customer_form.html └── tests/ ├── factories.py # factory-boy factories ├── test_models.py ├── test_views.py └── test_forms.py{% endcode %}
