The five edits

Adding a new page is a predictable, repeatable workflow. We'll add a "Reports" page under the existing apps/dashboard app:

  1. Add a CBV in the app's views.py
  2. Wire a URL in the app's urls.py
  3. Create the template under templates/<app>/
  4. Add a NavItem in apps/core/navigation.py (so the sidebar + palette pick it up)
  5. Refresh the browser — done

1. The view

{% code lang="python" %}# apps/dashboard/views.py from django.contrib.auth.mixins import LoginRequiredMixin from django.views.generic import TemplateView from apps.accounts.mixins import EmailVerifiedRequiredMixin from apps.accounts.views import StaffRequiredMixin from apps.core.breadcrumbs import BreadcrumbsMixin class ReportsView(BreadcrumbsMixin, LoginRequiredMixin, EmailVerifiedRequiredMixin, StaffRequiredMixin, TemplateView): template_name = "dashboard/reports.html" breadcrumb_title = "Reports" def get_context_data(self, **kwargs): ctx = super().get_context_data(**kwargs) ctx["report_data"] = {...} # whatever your page needs return ctx{% endcode %}

The mixin order matters — see the Architecture page.

5. Verify

  • Refresh the browser — the sidebar now lists "Reports" under Dashboards.
  • Press Cmd/Ctrl+K and type "reports" — the command palette finds it.
  • Click it — the page renders with the dashboard chrome (sidebar, header, breadcrumb).

Adding a model + CRUD

For a list/detail/create/edit/delete surface, the same pattern scales — look at any existing CRUD app (apps/customers/, apps/products/) for the recipe:

  1. Model + migration
  2. ModelForm in forms.py with our widget overrides
  3. Five CBVs: ListView (with TableView mixin), DetailView, CreateView, UpdateView, DeleteView
  4. Five templates extending layouts/dashboard.html
  5. One NavItem

The TableView mixin in apps/core/tables/ handles sortable columns, per-column filters, URL-state persistence, saved views, bulk actions, and CSV/XLSX export — all configured via class attributes.
