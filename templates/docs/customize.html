1. Brand name everywhere "Apex" appears

Search-and-replace covers most of it:

{% code lang="bash" %}# GNU sed (Linux): grep -rl --include='*.html' --include='*.py' --include='*.md' \ --exclude-dir=node_modules --exclude-dir=.venv 'Apex' . \ | xargs sed -i 's/Apex/Yourbrand/g' # macOS / BSD sed: change `-i` to `-i ''`{% endcode %}

After the bulk replace, verify these files manually:

  • templates/base.html — <title> block default
  • templates/partials/sidebar.html — sidebar logo word
  • apex/settings/base.py — DEMO_USERNAME / DEMO_PASSWORD
  • apps/marketing/templates/marketing/*.html — hero copy, footer, OG tags
  • templates/registration/password_reset_email.{txt,html} — email subject + signoff
{% callout type="warn" %} Don't rename the Python package apex/ itself unless you have a reason — the directory name is internal-only and renaming it cascades into DJANGO_SETTINGS_MODULE, WSGI_APPLICATION, ASGI_APPLICATION, and the systemd unit. {% endcallout %}

3. Color palette (design tokens)

All brand colors live in one file: static_src/css/input.css, under :root (light) and .dark (dark). They use OKLCh so any change keeps perceptual contrast clean across both modes. See the dedicated Theming guide.

4. Site URL + email defaults

The two most critical production env vars:

{% code lang="env" %}# /etc/yourapp.env on the server ALLOWED_HOSTS=app.yourbrand.com DEFAULT_FROM_EMAIL=Yourbrand # SMTP if you actually want password resets / invites to deliver: EMAIL_HOST=smtp.yourbrand.com EMAIL_PORT=587 EMAIL_HOST_USER=... EMAIL_HOST_PASSWORD=... EMAIL_USE_TLS=true{% endcode %}

Without these, password reset emails go to the console (fine for dev, useless in prod).

5. Cleanup before launch

  • Demo mode — set DEMO_MODE=false (or unset) in your env file.
  • seed_demo — keep for dev sample data, but never run on production. It creates a demo user with a known password.
  • CHANGELOG history — clear it out and start your own ## [1.0.0] line.
  • docs/superpowers/ — phase planning docs for how the template was built; not relevant to your fork.
