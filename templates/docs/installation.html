Prerequisites

  • Python 3.12 or 3.13 — NOT 3.14 (Django 5.1 incompatibility)
  • Node.js 20+ — for the Tailwind CLI
  • uv — install (or use pip)
  • WeasyPrint native libs — only if you want PDF invoice rendering

Clone + install dependencies

{% code lang="bash" %}git clone https://github.com/puikinsh/dashboardpack-apex-django.git cd dashboardpack-apex-django uv sync --all-groups # Python deps including dev/test npm install # Tailwind CLI + EasyMDE npm run build # Compile static/css/app.css{% endcode %}

Initialize the database

{% code lang="bash" %}uv run python manage.py migrate uv run python manage.py seed_demo{% endcode %}

The seed creates 15 users, 3 organizations, 100 customers, 25 products, 30 orders, 15 invoices spanning every status, 6 projects with milestones, 19 help-center articles, 9 blog posts, demo files, calendar events, kanban cards, mail threads, and chat history. The demo user is demo / ApexShowcase!2026.

Run the dev server

{% code lang="bash" %}uv run python manage.py runserver{% endcode %}

Open http://localhost:8000/. The login form pre-fills the demo credentials in dev mode — click Sign in and you're in. To turn that off: DEMO_MODE = False in apex/settings/dev.py.

Tailwind watch mode

For fast iteration on styles, run Tailwind in a second terminal:

{% code lang="bash" %}npm run dev{% endcode %} {% callout type="note" %} static/css/app.css is a build artifact and gitignored. Edit static_src/css/input.css where all design tokens live. {% endcallout %}

WeasyPrint native libs (optional)

Only needed if you want the invoice PDF download endpoint to actually work. Tests skip the PDF render path automatically when WeasyPrint can't load.

macOS

brew install pango

Ubuntu / Debian

sudo apt install libpango-1.0-0 libpangoft2-1.0-0 libcairo2

Run tests

{% code lang="bash" %}uv run pytest # ~1000 unit tests, under 90s uv run pytest -m e2e # Playwright E2E (spawns Chromium) ./run-e2e.sh # both, sequentially{% endcode %}

Common gotchas

  • No module named 'channels' — use uv sync not pip install; Channels is locked through uv.lock.
  • HTTP/2 support not enabled at Daphne start — benign notice. Daphne uses HTTP/1.1 to upstream; HTTP/2 works fine via nginx or Cloudflare.
  • Test DB stuck after a crash — rm db.sqlite3 && uv run python manage.py migrate && uv run python manage.py seed_demo
  • Stale CSS in browser — hard-refresh after npm run build. In prod, WhiteNoise's manifest hashing cache-busts automatically.
