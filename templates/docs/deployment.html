Architecture

{% code lang="text" %}Cloudflare ─▶ nginx ─▶ unix:/run/apex-django/apex-django.sock ─▶ Daphne ─▶ Django │ └─▶ /static/ ─▶ /var/www/apex-django/staticfiles/ (served directly){% endcode %}
  • Cloudflare — TLS termination, DDoS shield, CDN. Origin uses the shared *.yourdomain.com wildcard cert (or a Cloudflare Origin Certificate).
  • nginx — terminates origin TLS, serves /static/ and /media/ directly via alias, proxies everything else to Daphne over a unix socket.
  • Daphne — Django HTTP + WebSocket, single process, in apex-django.service systemd unit.
  • Postgres + Redis — co-located on the same box.

First-time setup (~15 min)

The complete walkthrough lives in deploy/README.md. Summary:

  1. DNS: A record for the subdomain → server IP, orange-cloud on, Cloudflare SSL = Full (strict).
  2. System packages: apt install postgresql redis-server libpango-1.0-0 libpangoft2-1.0-0 libcairo2 nodejs + curl -LsSf https://astral.sh/uv/install.sh | sh.
  3. Service user: useradd --system --create-home --home-dir /var/www/apex-django apex && usermod -aG www-data apex.
  4. Postgres: createuser apex_django -P && createdb apex_django -O apex_django.
  5. Env file: copy deploy/.env.example → /etc/apex-django.env, fill in SECRET_KEY + DATABASE_URL, chmod 0640 && chown apex:apex.
  6. First run: uv sync && npm ci && npm run build && python manage.py migrate && python manage.py collectstatic && python manage.py seed_demo.
  7. systemd: copy deploy/systemd/apex-django.service → /etc/systemd/system/, systemctl enable --now apex-django.
  8. nginx: copy the vhost into /etc/nginx/sites-available/, symlink to sites-enabled/, nginx -t && systemctl reload nginx.
  9. Smoke test: curl -I https://your-domain/accounts/login/ → 200.

Coexistence with PHP / Node sites

The new vhost only adds a new server { server_name your-domain; } block — existing vhosts are untouched. Watch out for:

  • map $http_upgrade $connection_upgrade { … } — declared once globally (we put it in /etc/nginx/conf.d/upgrade-map.conf). nginx errors on duplicate map declarations.
  • Postgres role isolation — give Apex its own apex_django role + DB; don't share with other apps.
  • Redis DB number — pick one (e.g. redis://127.0.0.1:6380/3) separate from whatever PHP/Node services are using.

Ongoing deploys

Once first-time setup is done:

{% code lang="bash" %}sudo -u apex /var/www/apex-django/deploy/scripts/deploy.sh{% endcode %}

Pulls, syncs deps, builds CSS, migrates, collects static, gracefully reloads Daphne. Open WebSocket sessions survive the reload.

{% callout type="tip" %} Set up a GitHub deploy key on the server (ssh-keygen -t ed25519 as the apex user, add public key to GitHub repo's Deploy keys) so git pull works without a password prompt. {% endcallout %}
