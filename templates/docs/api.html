Browse the API

Apex uses Django Ninja for its REST API. Open the running app at:

  • /api/v1/docs — Swagger UI for live exploration
  • /api/v1/openapi.json — raw OpenAPI 3 spec

Authentication

Bearer tokens issued via the Settings → Integrations → API tokens page. Stored as SHA-256 hashes (Stripe pattern) — the plaintext token is shown once on creation and never again. Pass on every request:

{% code lang="bash" %}curl https://apex-django.example.com/api/v1/customers/ \ -H "Authorization: Bearer apex_live_AbC1234..."{% endcode %}

Cursor pagination

All list endpoints use opaque cursor pagination — stable across inserts, no off-by-one risk. Response shape:

{% code lang="json" %}{ "items": [...], "next_cursor": "eyJpZCI6MTIzfQ==" // null when no more pages }{% endcode %}

Pass ?cursor=... on the next request to fetch the next page.

Adding an endpoint

Routers live in apps/api/api.py. Pattern:

{% code lang="python" %}from ninja import Router, Schema from apps.customers.models import Customer router = Router(tags=["customers"]) class CustomerOut(Schema): id: int name: str email: str @router.get("/customers/{id}", response=CustomerOut) def get_customer(request, id: int): return Customer.objects.get(pk=id){% endcode %}

Register the router with the main API instance — Swagger UI picks it up automatically.

Webhooks (HMAC-SHA256)

Apex can both send and receive HMAC-signed webhooks. Outgoing payloads use canonical JSON (sorted keys, no whitespace) so consumers can verify the signature deterministically:

{% code lang="python" %}# How consumers verify Apex's webhooks: import hmac, hashlib def verify(payload: bytes, signature: str, secret: str) -> bool: expected = hmac.new( secret.encode(), payload, hashlib.sha256, ).hexdigest() return hmac.compare_digest(expected, signature){% endcode %}

Webhook configurations + delivery log live under Settings → Integrations → Webhooks. Failed deliveries are retried with exponential backoff and surfaced in the delivery log for inspection.

Rate limits

No global rate limit ships out of the box, but the LoginThrottleMiddleware pattern in apps/accounts/throttle.py is easy to copy-adapt for API endpoints — see it for the cache-backed sliding-window approach.
