Consumers

Two consumers ship out of the box (in apps/realtime/consumers.py):

  • NotificationConsumer at /ws/notifications/ — per-user fan-out channel notify.user.<id>. Receives notify.message (new row) and notify.count (badge bump) events.
  • PresenceConsumer at /ws/presence/ — global presence channel. Tracks per-process connection set; broadcasts a count + the joined/left username on every transition.

Pushing a notification from server-side code

{% code lang="python" %}from apps.realtime.dispatch import push_notification, push_unread_count # Fire from any view, signal handler, or management command: push_notification(user.id, { "id": notification.id, "title": "Invoice paid", "body": "Acme Corp paid INV-2026-0001", "url": "/invoices/123/", }) push_unread_count(user.id, 7) # update bell badge across all open tabs{% endcode %}

The notification dispatcher (apps.notifications.dispatch.notify()) already calls these for you after every Notification.objects.create(). Every existing notification source (orders, invoices, mail, chat, mentions) becomes live with no per-call wiring.

Adding your own consumer

  1. Subclass AsyncJsonWebsocketConsumer in apps/realtime/consumers.py.
  2. Implement connect() + disconnect() + your group event handlers.
  3. Add a path() to apps/realtime/routing.py.
  4. Push from server-side via async_to_sync(layer.group_send)(...) — see apps/realtime/dispatch.py for the wrapper pattern.

Tests

Async consumer tests use WebsocketCommunicator + @pytest.mark.asyncio. See apps/realtime/tests/test_consumers.py for the full pattern (anonymous reject, accept, group fan-out, per-user isolation, presence count + join/leave broadcast).
