Pushing a toast

The most common cross-cutting primitive — call from any inline handler:

{% code lang="javascript" %}// Anywhere in your client code: window.apexToast({ level: "success", // "info" | "success" | "warning" | "error" body: "Changes saved.", action: { label: "View", href: "/x/" }, // optional CTA persistent: false, // default false (auto-dismisses after 5s) });{% endcode %}

Server-side, Django messages are converted to toasts automatically by the apex_toasts tag at the bottom of layouts/dashboard.html — just call messages.success(request, "...") in your view.

Adding your own primitive

Two pieces:

  1. An Alpine factory in static/js/shell.js exposing window.apexYourThing().
  2. A partial under templates/components/primitives/ with the markup that uses it via x-data="apexYourThing()".

Optionally add a documentation page at templates/components/pages/ with a live demo + code snippet — that's what powers the /components/ showcase.
