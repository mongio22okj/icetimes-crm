The pattern

Apex chart factories are Alpine components that own a single ApexCharts instance. They:

  1. Resolve theme colors from CSS custom properties at render time.
  2. Fetch JSON from a Django endpoint when the page loads.
  3. Render the chart.
  4. Use a MutationObserver on <html>'s class to detect dark-mode toggles and re-render with the new palette.

3. The Alpine factory

{% code lang="javascript" %}// static/js/charts.js window.reportsChart = function reportsChart() { let chart; function themeColors() { const css = getComputedStyle(document.documentElement); return { primary: css.getPropertyValue("--primary").trim(), muted: css.getPropertyValue("--muted-foreground").trim(), grid: css.getPropertyValue("--border").trim(), }; } function buildOptions(payload) { const colors = themeColors(); return { chart: { type: "line", height: 280, toolbar: { show: false } }, series: payload.series, xaxis: { categories: payload.labels, labels: { style: { colors: colors.muted } } }, colors: [colors.primary], stroke: { curve: "smooth", width: 2 }, grid: { borderColor: colors.grid }, }; } return { async init() { const r = await fetch("/dashboard/charts/reports/"); const data = await r.json(); chart = new ApexCharts(this.$el, buildOptions(data)); chart.render(); // Re-render on theme change const obs = new MutationObserver(() => { chart.updateOptions(buildOptions(data)); }); obs.observe(document.documentElement, { attributes: true, attributeFilter: ["class"] }); }, }; };{% endcode %}

Why this pattern?

ApexCharts renders SVG colors at chart-init time, so changing .dark after the chart is rendered doesn't touch the SVG. The MutationObserver pattern fires updateOptions() with a freshly-resolved palette every time the user toggles dark mode — fast, no extra deps, no re-fetch of the data.
