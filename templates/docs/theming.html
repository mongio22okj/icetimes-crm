Where colors live

Every color in Apex is a CSS custom property in static_src/css/input.css, declared once for light and once for dark:

{% code lang="css" %}:root { --background: oklch(1 0 0); --foreground: oklch(0.145 0 0); --primary: oklch(0.55 0.25 264); /* indigo */ --primary-foreground: oklch(0.985 0 0); --muted: oklch(0.97 0 0); --muted-foreground: oklch(0.556 0 0); --accent: oklch(0.97 0 0); --border: oklch(0.92 0 0); /* …sidebar-*, card-*, success, warning, destructive, etc. */ } .dark { --background: oklch(0.145 0 0); --foreground: oklch(0.985 0 0); --primary: oklch(0.65 0.25 264); --primary-foreground: oklch(0.145 0 0); /* …matched dark-mode pair for every light token */ }{% endcode %}

Why OKLCh?

OKLCh is perceptually uniform — changing the L (lightness) by the same amount looks like the same change across every hue. This means dark-mode colors that are "the same brightness" actually look that way to the eye, which is impossible to guarantee with hex or HSL. Tools like oklch.com let you pick by eye and copy the values straight into the CSS.

Recoloring step-by-step

  1. Pick a primary hue (e.g. 264 = indigo, 25 = orange-red).
  2. Edit --primary in :root with L=0.55, C=0.25, the new H.
  3. Edit --primary in .dark with L=0.65 (slightly brighter).
  4. Run npm run build.
{% callout type="tip" %} For a thorough recolor, change the L (lightness) and C (chroma) together in matched pairs across both :root and .dark so contrast stays balanced. {% endcallout %}

How charts pick up the theme

ApexCharts renders SVG colors at chart-init time, so changing .dark after render doesn't update the chart. Apex chart factories solve this with a MutationObserver on <html>'s class attribute that re-resolves theme colors and re-renders. See Charts for the full pattern.
