Models

    {% feature title="Organization" %} Name, auto-generated slug (with collision suffixing), optional logo, plan choice (free / pro / enterprise), created_by, timestamps. {% endfeature %} {% feature title="Membership" %} Joins User × Organization with a role, unique on (user, org), indexed for fast lookup. {% endfeature %} {% feature title="Invitation" %} Email + role + opaque secrets.token_urlsafe(32) token, 14-day TTL, accepted_at timestamp. Idempotent accept() creates the Membership and tolerates double-redeem. {% endfeature %}

Roles

Five roles, strict hierarchy from most- to least-privileged:

RoleCan do
ownerEverything, including delete the org
adminManage members + settings, can't delete the org
billingManage subscriptions + payment methods, see invoices
memberRead + write tenant-scoped data
viewerRead-only access to tenant-scoped data

Request middleware

OrganizationMiddleware resolves request.organization, request.memberships, request.organization_role for every authenticated request. Precedence:

  1. session-stored slug → 2. first membership alphabetically → 3. None.

Failures are swallowed so a tenant lookup never breaks the request.

UI surfaces

  • /orgs/ — list of memberships with switch / settings actions + create-new form (owner role auto-assigned).
  • /orgs/<slug>/settings/ — name + plan editor with an owner-only Danger Zone delete confirmation.
  • /orgs/<slug>/members/ — member roster with inline role change + remove buttons (admin/owner only). Pending invitations panel with expiry countdown and cancel action.
  • /invitations/<token>/ — public accept page, handles unauthenticated, wrong-account, expired, already-accepted, and happy-path states.
  • Header org switcher — auth-only dropdown showing all memberships.

Where to read more

apps/organizations/ is self-contained — start with models.py (the role_at_least() helper is the heart of the RBAC), then mixins.py for the view integration. Tests in apps/organizations/tests/ show every flow end-to-end.
