No SPA

Apex is built for product teams that ship features over rebuilds. Server-rendered Django + HTMX + Alpine gets you ~95% of what an SPA gives a typical dashboard, with ~10% of the moving parts. There is no React, no Vue, no Svelte, no build step beyond Tailwind. HTMX handles the swappable bits (datatable filters, kanban drag, mail thread reply) via attributes; Alpine handles the local interactive bits (dropdowns, modals, toasts) via x-data. Total client-side JS is ~30KB across every page.

Navigation = single source of truth

Don't add sidebar items by editing templates. All nav lives in apps/core/navigation.py as a NAV_ITEMS tuple of NavItem dataclasses (label, url_name, icon, group, requires_staff). The navigation context processor feeds both:

  • The grouped sidebar (partials/sidebar.html)
  • The command palette (Cmd/Ctrl+K) via json_script

requires_staff=True filters items out for non-staff users in both surfaces.

Auth mixin stack (order matters)

Protected CBVs typically compose mixins in this order:

{% code lang="python" %}class FooView(BreadcrumbsMixin, LoginRequiredMixin, EmailVerifiedRequiredMixin, StaffRequiredMixin, PasswordConfirmationRequiredMixin, UpdateView): ...{% endcode %}
  • LoginRequiredMixin first — unauthenticated users redirect to login.
  • EmailVerifiedRequiredMixin — unverified users redirect to email-verify.
  • StaffRequiredMixin — non-staff users get 403 (vs. redirect).
  • PasswordConfirmationRequiredMixin — re-prompts for password on sensitive actions; 3-hour grace via session.
  • BreadcrumbsMixin — injects a breadcrumbs context var.

For org-scoped routes, add OrgRequiredMixin and/or HasRoleMixin from apps.organizations.mixins before the data-fetching mixins.

Design tokens (Tailwind v4)

All colors are OKLCh custom properties in static_src/css/input.css under :root (light) and .dark (dark) blocks. The @theme inline block exposes them as Tailwind utilities (bg-background, text-sidebar-accent-foreground, etc.). Dark mode is class-based; an inline script in templates/base.html applies .dark before paint to prevent flash.

{% callout type="tip" %} Edit input.css, then npm run build — static/css/app.css is the gitignored build artifact. {% endcallout %}

Charts (theme-aware)

static/js/charts.js defines Alpine factories like revenueChart(). Each chart resolves theme colors at render time and uses a MutationObserver on <html>'s class attribute to re-render when the theme toggles. New charts should follow the same factory + observer pattern.

Realtime (Channels)

apex/asgi.py routes HTTP through Django and WebSocket through apps/realtime/routing.py consumers, wrapped in AuthMiddlewareStack + AllowedHostsOriginValidator. Two consumers ship out of the box:

  • NotificationConsumer (/ws/notifications/) — per-user fan-out.
  • PresenceConsumer (/ws/presence/) — global online users count.

Server-side code calls apps.realtime.dispatch.push_notification(user_id, payload) and the channel layer fans it out to every open tab the user has.
