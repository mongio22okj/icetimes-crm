What demo mode gives you

Setting DEMO_MODE=true in the production env file flips two visible behaviors:

  • Login auto-fill — the /accounts/login/ page pre-fills demo / ApexShowcase!2026 and shows a "Demo credentials" banner.
  • Demo banner — a sticky orange bar at the top of every dashboard page reminding visitors this is a demo where data resets. Dismissable per-tab via sessionStorage.

Both gate on settings.DEMO_MODE, so a real production deploy with the var unset shows neither.

The reset_demo command

apps/core/management/commands/reset_demo.py flushes the database and re-runs seed_demo to restore the canonical sandbox.

{% code lang="bash" %}uv run python manage.py reset_demo # confirms y/N first uv run python manage.py reset_demo --no-input # for cron{% endcode %} {% callout type="warn" %} Safety belt: refuses to run unless settings.DEMO_MODE=True (or --force), so it can never accidentally wipe a real production database. {% endcallout %}

Hourly cron — the recommended setup

Install for the apex user:

{% code lang="bash" %}sudo crontab -u apex -e{% endcode %}

Add:

{% code lang="cron" %}# Reset demo data every hour at HH:05 5 * * * * /var/www/apex-django/deploy/scripts/reset_demo_cron.sh >> /var/log/apex-django/reset.log 2>&1{% endcode %}

Make sure the log directory exists and is writable by the apex user:

{% code lang="bash" %}sudo install -d -o apex -g apex -m 0755 /var/log/apex-django{% endcode %}

The wrapper script (deploy/scripts/reset_demo_cron.sh) sources /etc/apex-django.env so DJANGO_SETTINGS_MODULE, DATABASE_URL, etc. are available — cron runs in a minimal environment without these by default.

Full reset takes ~5 seconds on a small VPS. Visitors signed in during the reset get logged out the next time they make a request.

Verifying the reset worked

{% code lang="bash" %}# After the cron fires, hit the health endpoint: curl https://apex-django.example.com/__health/ # {"status":"ok","checks":{"db":{"ok":true,"ms":1.2}, …}}{% endcode %}

The bell badge shows ~30 unread notifications fresh out of seed_demo.
